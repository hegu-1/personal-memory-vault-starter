# Contributing

Thanks for your interest. This is a **starter template** — most people will *use* it (clone, or click "Use this template") rather than contribute back. But improvements to the template itself are very welcome.

## Ways to contribute

- **Improve the docs** — clearer wording in the README, the guides, or the directory READMEs
- **Better examples** — the `example-*.md` files should stay generic, fictional, and instructive
- **New node types or conventions** — open an issue first so we can discuss the design
- **Translations** — the repo is English-primary with a Chinese on-ramp (`HUMAN_GUIDE.md`); other-language on-ramps are welcome as `HUMAN_GUIDE.<lang>.md`

## Ground rules

- Keep example content **fully fictional and generic** — no real people, companies, or private data.
- Don't add large binaries (PDFs, videos). Store metadata + links only (see `resources/`).
- Keep the format tool-agnostic: plain markdown + YAML frontmatter, no editor-specific syntax.
- One logical change per PR; write descriptive commit messages.

## Workflow

1. Fork the repo
2. Create a branch: `git checkout -b improve-x`
3. Make your change and commit with a clear message
4. Open a PR describing *what* and *why*

By contributing, you agree your contributions are released under **CC0 1.0** (public domain), the same as the rest of the repo.
