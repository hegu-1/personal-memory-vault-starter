# concepts/

存抽象概念 / 框架 / 思想体系 / 你信的某种道理。

## 什么时候用

- 你有一套对某个 topic 的看法
- 某个理论 / 框架是你思考的 anchor
- 一个术语 / 概念需要被反复引用

## 命名约定

- `<concept-name>.md`,kebab-case
- 例:`time-management.md` / `my-view-of-ai.md` / `nepm.md`

## 跟 playbooks/ 区别

- **concept**:抽象的看法 / 框架 / 理论
- **playbook**:具体的操作 / 规则 / how-to

如果一个 file 既描述思想又给操作步骤,默认按"主要 framing"分:抽象主导 → concepts,操作主导 → playbooks。
